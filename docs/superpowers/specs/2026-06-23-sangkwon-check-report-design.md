# 상권 체크 리포트 — 설계 문서

**날짜:** 2026-06-23  
**상태:** 승인됨

---

## 1. 제품 개요

지역명 + 업종을 입력하면 네이버 검색 API로 주변 경쟁 매장 데이터를 수집하고, Gemini AI가 포지셔닝 분석 코멘트를 생성해주는 Streamlit 대시보드 앱.

**목표:** 예비 창업자가 3분 이내에 상권 분석 결과를 확인하고 의사결정에 참고한다.

---

## 2. 기술 스택 (전부 무료)

| 역할 | 도구 |
|------|------|
| 앱 화면 | Streamlit |
| 가게 데이터 수집 | 네이버 검색 API |
| AI 분석 | Gemini API (gemini-1.5-flash) |
| 코드 저장 | GitHub |
| 인터넷 배포 | Streamlit Community Cloud |

- Google Sheets 제외 (불필요한 복잡성)
- Selenium 크롤링 제외 (차단 위험, 유지보수 어려움)

---

## 3. 화면 구성 (단일 페이지)

위에서 아래 순서:

1. **검색창** — "연남동 카페" 형식 입력 + "리포트 생성" 버튼
2. **로딩 애니메이션** — 분석 중 스피너 표시
3. **요약 카드 3개** — 평균 평점 / 분석 매장 수 / 최다 카테고리
4. **경쟁 매장 표** — 상호명, 주소, 카테고리 (상위 15개)
5. **AI 분석 코멘트** — 경쟁 강도 평가 + 포지셔닝 제안 + 추천/비추천 결론

---

## 4. 데이터 흐름

```
사용자 입력 ("연남동 카페")
        ↓
네이버 검색 API 호출
→ 상위 15개 매장: 이름, 주소, 카테고리, 전화번호
        ↓
Gemini API 호출
→ 프롬프트: 수집된 매장 목록 + "경쟁 강도, 포지셔닝 전략, 추천 여부 분석"
→ 응답: 분석 텍스트
        ↓
Streamlit 화면에 표·카드·AI 코멘트 표시
(데이터 저장 없음, 매번 새로 분석)
```

---

## 5. 파일 구조

```
Claude-Niche-Finder-app/
├── app.py              # 메인 Streamlit 앱
├── naver_api.py        # 네이버 검색 API 호출
├── gemini_api.py       # Gemini AI 분석
├── requirements.txt    # 필요한 패키지 목록
└── .streamlit/
    └── secrets.toml    # API 키 저장 (GitHub에 올리지 않음)
```

---

## 6. 에러 처리

- API 키 누락 시: 화면에 "API 키를 설정해주세요" 안내 메시지 표시
- 검색 결과 없음 시: "해당 지역/업종의 매장을 찾을 수 없습니다" 표시
- API 호출 실패 시: 재시도 안내 메시지 표시

---

## 7. 제약사항

- 네이버 검색 API: 하루 25,000건 무료 (가끔 사용 기준 충분)
- Gemini API: 무료 티어 분당 15회 제한 (가끔 사용 기준 충분)
- 리뷰 텍스트 수집 불가 (네이버 공식 API 미지원) → 매장 목록 기반 AI 분석으로 대체
- Streamlit Community Cloud 동시 접속자 제한 있음 (소수 지인 사용 기준 충분)
